# Club Guide

This page gives practical advice for using Nexgit as a university club project.

## Goals for the club

Nexgit should help members learn:

- Open-source collaboration.
- Git and GitHub workflows.
- Rust application development.
- TypeScript desktop development.
- Product and design thinking.
- Code review and technical communication.

## Beginner task ladder

Suggested progression for new members:

1. Run the project locally.
2. Fix or improve docs.
3. Change CLI help text.
4. Improve TUI placeholder text/layout.
5. Add a simple app-server method.
6. Add a desktop UI section for existing data.
7. Work on real Git behavior.

## Pairing ideas

Pair newer and more experienced contributors on:

- First Rust PR.
- First Electron/React PR.
- Debugging app-server messages.
- Writing tests.
- Reviewing a PR together.

## Semester planning

At the start of a semester:

- Pick 1-2 realistic milestones.
- Identify beginner-friendly issues.
- Confirm maintainers and project leads.
- Review docs for stale setup steps.

At the end of a semester:

- Write a short status update.
- Archive or close stale issues.
- Update milestone priorities.
- Confirm who will maintain the project next.

## Keeping the project healthy

- Prefer small PRs.
- Avoid private-only decisions; summarize in issues or docs.
- Celebrate first contributions.
- Do not let the codebase become understandable only to the original authors.
- Keep setup steps accurate.
